Shared Embeddings in NLP

14 czerwca 2024

Co oznacza poniższy tekst? "Obowiązują shared embeddings (współdzielone zanurzenia), a zatem macierz przetwarzająca one-hot wektory alfabetu na gęstą reprezentację powinna być transpozycją macierzy tłumaczącej wyjście modelu na logitsy."

Tekst ten odnosi się do koncepcji związanej z modelami przetwarzania języka naturalnego (NLP) i głębokiego uczenia się, w szczególności do sposobu reprezentacji słów i przetwarzania ich przez model.

  1. Shared embeddings (współdzielone zanurzenia): W modelach NLP, embeddingi (zanurzenia) to sposób reprezentowania słów jako wektorów w przestrzeni wielowymiarowej. "Współdzielone embeddingi" oznaczają, że ta sama macierz embeddingów jest używana zarówno w warstwie wejściowej, jak i wyjściowej modelu.

  2. Macierz przetwarzająca one-hot wektory alfabetu na gęstą reprezentację: One-hot wektory to sposób reprezentacji słów, gdzie każde słowo jest reprezentowane jako wektor z jedną jedynką i resztą zer. Macierz przetwarzająca one-hot wektory na gęstą reprezentację (embeddingi) konwertuje te wektory do bardziej zwartej, wielowymiarowej formy, co ułatwia modelowi operowanie na danych.

  3. Powinna być transpozycją macierzy tłumaczącej wyjście modelu na logitsy: Logitsy to wartości wyjściowe modelu przed zastosowaniem funkcji softmax, która konwertuje je na prawdopodobieństwa. Stwierdzenie to oznacza, że macierz używana do przekształcania one-hot wektorów na embeddingi powinna być transponowaną wersją macierzy używanej do przekształcania wyjścia modelu (embeddingów) z powrotem na logitsy. Innymi słowy, obie macierze są wzajemnie transponowane.

Podsumowując, w tekście chodzi o to, że w modelu NLP, gdy używa się wspólnych embeddingów, macierz używana do tworzenia embeddingów z one-hot wektorów powinna być transponowaną wersją macierzy używanej do przekształcania wyjścia modelu na logitsy.

Pokaż tę koncepcję w torch. Załóżmy że chcemy użyć LSTM do modelowania języka

Aby zilustrować tę koncepcję w PyTorchu, załóżmy, że chcemy użyć LSTM do modelowania języka z shared embeddings. Poniżej znajduje się przykładowy kod, który pokazuje, jak to zrobić:

python
import torch
import torch.nn as nn

# Ustalmy parametry modelu
vocab_size = 10000  # Rozmiar słownika
embedding_dim = 300  # Wymiar embeddingów
hidden_dim = 512  # Wymiar ukrytej warstwy LSTM
num_layers = 2  # Liczba warstw LSTM

class LanguageModel(nn.Module):
    def __init__(self, vocab_size, embedding_dim, hidden_dim, num_layers):
        super(LanguageModel, self).__init__()
        self.embedding = nn.Embedding(vocab_size, embedding_dim)
        self.lstm = nn.LSTM(embedding_dim, hidden_dim, num_layers, batch_first=True)
        self.fc = nn.Linear(hidden_dim, vocab_size)

        # Inicjalizacja shared embeddings
        self.fc.weight = nn.Parameter(self.embedding.weight.t())

    def forward(self, x, hidden):
        embedded = self.embedding(x)
        lstm_out, hidden = self.lstm(embedded, hidden)
        logits = self.fc(lstm_out)
        return logits, hidden

    def init_hidden(self, batch_size):
        weight = next(self.parameters()).data
        return (weight.new(self.lstm.num_layers, batch_size, self.lstm.hidden_size).zero_(),
                weight.new(self.lstm.num_layers, batch_size, self.lstm.hidden_size).zero_())

# Przykładowe użycie modelu
batch_size = 32
seq_length = 20

model = LanguageModel(vocab_size, embedding_dim, hidden_dim, num_layers)
hidden = model.init_hidden(batch_size)

# Przykładowe dane wejściowe (losowe indeksy ze słownika)
input_data = torch.randint(0, vocab_size, (batch_size, seq_length), dtype=torch.long)

# Forward pass
logits, hidden = model(input_data, hidden)

# Wyniki
print(logits.shape)  # Powinno dać (batch_size, seq_length, vocab_size)

Wyjaśnienie kodu:

  1. Warstwa embeddingu: Tworzymy warstwę embeddingu nn.Embedding, która zamienia one-hot wektory na gęste wektory embeddingowe.
  2. LSTM: Używamy warstwy LSTM nn.LSTM do przetwarzania sekwencji embeddingów.
  3. Warstwa w pełni połączona (fully connected): nn.Linear służy do mapowania wyjścia LSTM na przestrzeń logitsów. Waga tej warstwy jest ustawiona na transponowaną wersję macierzy wagowej warstwy embeddingu.
  4. Inicjalizacja ukrytego stanu: Metoda init_hidden inicjalizuje ukryty stan LSTM.
  5. Przekazywanie danych przez model: W forward passie embeddingi są przekazywane przez LSTM, a następnie przekształcane na logitsy za pomocą warstwy w pełni połączonej.

Dzięki temu używamy shared embeddings, gdzie macierz wagowa warstwy embeddingu jest transpozycją macierzy wagowej warstwy w pełni połączonej, co realizuje opisaną koncepcję.

Załóżmy że mamy one-hot encoded 40 znaków. Przedstaw macierz, która posłuży do zagęszczenia reprezentacji. Następnie zagęszczoną reprezentację chcemy za pomocą innej macierzy przekształcić na 40 logitów, które będą kodowały konkretny znak z pewnym prawdopodobieństwem.

Aby zademonstrować ten proces, załóżmy, że mamy 40 znaków, które są one-hot encoded. Oznacza to, że każde wejście jest wektorem o wymiarze 40, gdzie tylko jedna z pozycji jest równa 1, a reszta to 0.

Zaczniemy od utworzenia macierzy embeddingowej (zagęszczającej reprezentację) oraz macierzy przekształcającej zagęszczoną reprezentację z powrotem na logity.

Poniżej znajduje się przykładowy kod w PyTorchu:

python
import torch
import torch.nn as nn

# Parametry
num_characters = 40  # Liczba znaków (rozmiar słownika)
embedding_dim = 10   # Wymiar embeddingów

# Macierz embeddingowa
embedding_matrix = nn.Parameter(torch.randn(num_characters, embedding_dim))

# Macierz przekształcająca embeddingi na logity
logit_matrix = nn.Parameter(torch.randn(embedding_dim, num_characters))

# Funkcja przetwarzająca one-hot encoded wektory na zagęszczoną reprezentację
def embed(one_hot_vectors, embedding_matrix):
    return one_hot_vectors @ embedding_matrix

# Funkcja przekształcająca zagęszczoną reprezentację na logity
def to_logits(embedded_vectors, logit_matrix):
    return embedded_vectors @ logit_matrix

# Przykładowe dane wejściowe (one-hot encoded wektory)
one_hot_vectors = torch.eye(num_characters)  # Macierz jednostkowa jako przykładowe one-hot encoded wektory

# Przetwarzanie one-hot encoded wektorów na zagęszczoną reprezentację
embedded_vectors = embed(one_hot_vectors, embedding_matrix)

# Przekształcanie zagęszczonej reprezentacji na logity
logits = to_logits(embedded_vectors, logit_matrix)

# Wyniki
print("Embedding Matrix:")
print(embedding_matrix)
print("\nEmbedded Vectors:")
print(embedded_vectors)
print("\nLogit Matrix:")
print(logit_matrix)
print("\nLogits:")
print(logits)

Wyjaśnienie kodu:

  1. Embedding Matrix: Macierz embedding_matrix o wymiarach 40×1040 \times 1040×10, która zamienia one-hot encoded wektory na zagęszczoną reprezentację (embeddingi).
  2. Logit Matrix: Macierz logit_matrix o wymiarach 10×4010 \times 4010×40, która zamienia embeddingi na logity.
  3. Embedding Function: Funkcja embed, która przetwarza one-hot encoded wektory na zagęszczoną reprezentację przez mnożenie macierzy.
  4. Logit Function: Funkcja to_logits, która przetwarza embeddingi na logity przez mnożenie macierzy.
  5. Example Input Data: one_hot_vectors to macierz jednostkowa o wymiarach 40×4040 \times 4040×40, która reprezentuje one-hot encoded wektory dla 40 znaków.

Wynikowy kod przekształca one-hot encoded wektory na embeddingi, a następnie na logity, które można użyć do przewidywania znaków z pewnym prawdopodobieństwem.

Co oznacza operator @? Czym jest nn.Parameter?

Operator @ w PyTorchu

Operator @ w PyTorchu (oraz w Pythonie 3.5 i nowszych) jest operatorem mnożenia macierzy. Działa on jako bardziej czytelna i zwięzła alternatywa dla funkcji torch.matmul lub np.matmul w NumPy.

Przykład użycia operatora @:

python
import torch

# Dwie macierze
A = torch.tensor([[1, 2], [3, 4]])
B = torch.tensor([[5, 6], [7, 8]])

# Mnożenie macierzy
C = A @ B

print(C)

Wynik:

lua
tensor([[19, 22],
        [43, 50]])

Tutaj operator @ mnoży macierze A i B, co daje macierz wynikową C.

Czym jest nn.Parameter?

nn.Parameter w PyTorchu jest specjalną klasą, która jest podklasą torch.Tensor. Parametry modeli definiowane jako nn.Parameter są automatycznie dodawane do listy parametrów modelu i będą podlegały optymalizacji podczas procesu treningowego.

Gdy tworzymy warstwy w modelu i przypisujemy im parametry, np. wagi i biasy, używamy nn.Parameter, aby PyTorch wiedział, że są to parametry, które powinny być aktualizowane podczas backpropagacji.

Przykład użycia nn.Parameter:

python
import torch
import torch.nn as nn

class MyModel(nn.Module):
    def __init__(self):
        super(MyModel, self).__init__()
        # Definiowanie parametru jako nn.Parameter
        self.my_param = nn.Parameter(torch.randn(2, 2))

    def forward(self, x):
        return x @ self.my_param

# Inicjalizacja modelu
model = MyModel()

# Wyświetlenie parametru
print(model.my_param)

Wynik:

lua
Parameter containing:
tensor([[ 1.2345, -0.6789],
        [ 0.1234,  0.9876]], requires_grad=True)

W powyższym przykładzie:

  1. Definiowanie parametru: self.my_param = nn.Parameter(torch.randn(2, 2)) tworzy macierz 2×22 \times 22×2 wypełnioną losowymi liczbami.
  2. Optymalizacja parametru: Ponieważ self.my_param jest instancją nn.Parameter, PyTorch będzie automatycznie śledził ten parametr podczas obliczeń wstecznych i aktualizował go podczas optymalizacji.

Zastosowanie w naszym kontekście

W naszym poprzednim przykładzie używamy nn.Parameter do zdefiniowania macierzy embeddingowej i macierzy przekształcającej embeddingi na logity. Dzięki temu te macierze będą optymalizowane podczas treningu modelu:

python
embedding_matrix = nn.Parameter(torch.randn(num_characters, embedding_dim))
logit_matrix = nn.Parameter(torch.randn(embedding_dim, num_characters))

To pozwala modelowi na uczenie się optymalnych wartości tych macierzy, aby lepiej reprezentować dane wejściowe i przewidywać wyniki.
